项目经历

  • 端到端语音识别建模单元的对比：对于CTC、Attention这两类端到端的方法，分别对比了音素、音节、字不同建模的性能；实验发现了一些有趣的结果，对不同条件下的语音识别应用有借鉴意义。发表论文

  • 端到端语音识别系统：基于LAS搭建端到端的语音识别，将多任务学习应用到模型训练，联合Attention与CTC，提升系统训练的鲁棒性，系统达到了较高的性能；并且部署上线，落地到业务。

  • 面向车载环境的语音识别与交互：针对车载环境中的噪声、混响等复杂情况，分别针对性地优化，大幅提升了车载语音识别性能，相对提升约50%。

  • 大数据条件下基于神经网络的语音合成及在明星音合成中的应用：在大规模语音数据条件下（几十甚至上百小时数据），高效准确地完成模型训练，并合成高质量的语音。通过模型自适应等技术，实现明星小数据量条件下的高质量语音合成。

  • 面向语音合成的数据清洗：语音合成对数据质量要求很高，面对海量数据，人工筛选更加不现实。利用语音识别技术识别语音数据并将识别结果与原始数据标注比对，计算相似度得分，将得分小于阈值的数据过滤。

  • 基于深层神经网络的语音合成：利用深层循环神经网络实现从文本特征到语音声学特征的映射，从而取代了传统基于HMM的语音合成架构中的决策树和高斯模型，大幅提高了声学模型性能。

  • 基于MGE的语音合成： 基于HMM的参数语音合成中，将模型训练准则由最大似然准则（MLE） 改为最小生成误差准则（MGE），提高了声学模型性能。

  • 电影检索系统：系统通过豆瓣API获取大量电影结构化信息；对原始电影信息做预处理并创建倒排索引；使用Perl和GTK2实现了该系统，可输入关键词检索相关电影。Source Code

  • 语音检索系统：首先，从多媒体数据中抽取语音，对语音数据标注得到标注文档；然后对标注文档构建索引，得到索引库；用户通过文本输入或者语音输入方式，提出查询请求；通过查询分析，从索引库中得到对应的文档集；最后将文档映射到多媒体数据中，返回给用户。利用Apache Tomcat搭建服务器，使用SQL Server搭建数据库，使用Javascript、CSS实现用户交互页面的设计。
